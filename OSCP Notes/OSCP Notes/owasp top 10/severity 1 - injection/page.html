About Injection :-
  • User controlled input is interpreted as actual commands or parameters.
  • Common examples include: SQL Injection, Command Injection
  • Happens due to lack sanitized input.
Defence :-
  • Using an allow list -  when input is sent to the server, this input is compared to a list of safe input or characters. If the input is marked as safe, then it is processed. Otherwise, it is rejected and the application throws an error.
  • Stripping input: If the input contains dangerous characters, these characters are removed before they are processed.
  • Can also use other libraries

- Dangerous characters or input is classified as any input that can change how the underlying data is processed.
